# Identity Lens
> Generated by AgentCompany on 2026-04-14

**Status**: complete
**Industry**: B2B SaaS developer tools

## Value Proposition
For security teams at mid-market SaaS companies, Identity Lens automatically discovers and visualizes the risk of non-human identities and API permissions across SaaS-to-SaaS integrations, focusing exclusively on the permissions graph rather than logs or user behavior.

## Customer Segment
**Security Engineer at SaaS Scale-ups** — A security engineer or lead at a B2B SaaS company with 50-500 employees, responsible for securing a growing tech stack but lacking dedicated AppSec or IAM specialists. They manage integrations like Salesforce, HubSpot, Slack, GitHub, and cloud providers.
Job-to-be-done: Quickly understand which third-party SaaS integrations have excessive API permissions and identify non-human service accounts that could be compromised, without manually auditing each vendor's admin console.

## MVP Features
- [Must] **SaaS Connector (Read-Only)**: A single, lightweight agent that connects to one core SaaS platform (e.g., GitHub, Slack) via OAuth with read-only scope. It extracts ONLY the list of installed third-party integrations/apps and their granted OAuth permission scopes.
- [Should] **Permission Risk Visualization**: A single dashboard view showing a force-directed graph. Nodes are: 1) the customer's core SaaS platform, and 2) each connected third-party app. Edges are labeled with the granted permission scopes. Scopes are color-coded (green/yellow/red) based on a static, pre-defined risk library (e.g., 'read:user' = low, 'admin:write' = high). No historical data.
- [Nice] **Non-Human Identity Tagger**: Automatically flags integrations that use service accounts or API keys (non-human identities) based on the app's authentication metadata. Presents a simple list: 'These 3 integrations use service accounts with high-risk permissions.'
- [Nice] **One-Click Report**: Generates a single-page PDF snapshot of the graph and the high-risk non-human identities list, formatted for sharing in a compliance or vendor review meeting.
